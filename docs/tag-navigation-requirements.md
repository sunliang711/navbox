# 导航站 Tag 组织需求文档

## 一、背景

导航站用于集中管理网站入口。每个网站以卡片形式展示，访问者可以通过图标、标题、描述、URL、打开方式、卡片样式等信息快速识别和访问目标网站。

为提升网站分类、筛选和管理效率，系统需要基于 `Tag` 组织网站。一个网站可以绑定多个 Tag，访问者通过 Tag 组合筛选网站入口，从而提升分类灵活性和查找效率。

系统支持两种访问方式：

- 游客：只读访问，用于查看、搜索、筛选和打开网站。
- admin：管理访问，用于维护网站、Tag、默认视图和展示配置。

## 二、目标

1. 使用 Tag 作为网站的主要组织方式。
2. 一个网站支持绑定多个 Tag。
3. 首页默认进入一个默认 Tag 视图。
4. 多 Tag 筛选规则为：同时包含所有选中 Tag。
5. Tag 支持颜色、图标、排序。
6. 支持游客和 admin 两种访问方式。
7. 游客只读，不能新增、编辑、删除或保存任何配置。
8. admin 负责网站、Tag、卡片样式、默认视图等管理操作。
9. admin 接口权限基于 Session 校验。
10. 支持配置导入和导出，导出范围可选择全部或指定数据。

## 三、核心概念

### 3.1 网站 Site

网站是具体的导航入口，例如 Gitlab、Grafana、Jenkins、Redis。

主要字段：

| 字段 | 说明 |
| --- | --- |
| 标题 | 网站名称，必填 |
| 描述 | 网站说明、账号提示或补充信息 |
| 默认 URL | 默认打开地址，必填 |
| LAN URL | 内网环境下的访问地址 |
| 图标类型 | 支持文本、图片、在线图标 |
| 图标地址 | 图片地址或在线图标地址 |
| 打开方式 | 当前窗口、新窗口等 |
| 卡片背景色 | 网站卡片背景色 |
| 是否仅显示名称 | 控制卡片展示模式 |
| 绑定 Tag 列表 | 网站所属 Tag，支持多个 |
| 是否常用 | 是否展示在常用视图 |
| 排序值 | 控制网站卡片展示顺序 |
| 创建时间 | 网站创建时间 |
| 更新时间 | 网站最近更新时间 |

### 3.2 Tag

Tag 是网站的组织方式。

主要字段：

| 字段 | 说明 |
| --- | --- |
| 名称 | Tag 名称，必填 |
| 图标 | Tag 图标 |
| 颜色 | Tag 颜色 |
| 排序值 | 控制 Tag 展示顺序 |
| 类型 | 系统 Tag、业务 Tag |
| 是否默认 Tag | 是否作为默认进入视图 |
| 是否启用 | 控制 Tag 是否可用 |

示例 Tag：

- `Tools`
- `Dev`
- `监控`
- `数据库`
- `CI/CD`
- `生产`
- `测试`
- `账号`
- `安全`
- `常用`

### 3.3 访问方式

系统支持 `游客` 和 `admin` 两种访问方式。

游客能力：

1. 查看网站列表。
2. 查看 Tag 列表。
3. 使用 Tag 筛选网站。
4. 使用关键字搜索网站。
5. 打开默认 URL。
6. 打开 LAN URL。
7. 复制链接。

admin 能力：

1. 拥有游客全部能力。
2. 新增、编辑、删除网站。
3. 新增、编辑、删除 Tag。
4. 设置网站绑定的 Tag。
5. 设置网站是否常用。
6. 设置默认 Tag。
7. 设置 Tag 颜色、图标和排序。
8. 上传或配置网站图标。
9. 设置网站卡片背景色和展示方式。
10. 修改 admin 密码。
11. 批量添加 Tag。
12. 批量移除 Tag。
13. 批量删除网站。
14. 调整网站展示顺序。
15. 导入和导出配置。

### 3.4 admin 认证与 Session

admin 认证要求：

1. 系统首次启动且未配置 admin 密码时，自动生成一个随机初始密码。
2. 初始密码需要打印到启动日志中，日志内容使用英文，例如 `Admin initial password: <password>`。
3. admin 登录成功后创建 Session。
4. admin 写接口必须校验有效 Session。
5. admin 可在登录后修改密码。
6. 修改密码成功后，建议使当前 Session 以外的其他 admin Session 失效。
7. admin 可主动退出登录，退出后当前 Session 失效。

## 四、首页展示

首页按 Tag 筛选展示网站入口。

默认行为：

1. 访问者进入首页后，自动选中默认 Tag。
2. 默认 Tag 由 admin 配置。
3. 如果默认 Tag 不存在或已禁用，则展示系统兜底视图 `全部`。

多 Tag 筛选规则：

```text
选中 Tag：Dev + 监控
展示结果：同时包含 Dev 和 监控 的网站
```

未绑定任何 Tag 的网站：

1. 归入系统视图 `未分类`。
2. admin 编辑网站时允许不绑定 Tag，但需要在列表中可被明确找到。
3. admin 新建网站时建议至少绑定一个 Tag。

## 五、Tag 导航区

页面需要提供 Tag 导航区，可放在顶部或左侧，具体布局以后续 UI 设计为准。

Tag 展示内容：

- 图标
- 名称
- 颜色标识
- 网站数量
- 选中状态

交互能力：

1. 点击 Tag：选中该 Tag。
2. 再次点击已选中 Tag：取消选中。
3. 支持多选 Tag。
4. 多选时，网站列表只展示同时包含所有选中 Tag 的网站。
5. 支持按排序值展示 Tag。
6. Tag 较多时支持搜索、折叠或滚动。

系统内置视图：

- `全部`
- `默认`
- `常用`
- `最近访问`
- `未分类`

系统内置视图作为筛选入口处理。

## 六、网站卡片

网站卡片需要支持以下展示能力：

- 图标
- 标题
- 描述或账号信息
- 背景色
- 默认 URL 打开
- LAN URL 打开
- 绑定的 Tag
- 常用标识

卡片上的 Tag 展示规则：

1. 默认最多显示 2 个 Tag。
2. 超出时显示 `+N`。
3. 鼠标悬停时展示完整 Tag 列表。
4. 移动端可通过点击或长按查看完整 Tag 列表。

游客模式下，卡片只展示只读操作。

admin 模式下，卡片展示管理操作。

## 七、卡片操作菜单

游客模式下，卡片操作菜单支持：

- 打开默认 URL
- 打开 LAN URL
- 复制链接

admin 模式下，卡片操作菜单支持：

- 打开默认 URL
- 打开 LAN URL
- 复制链接
- 编辑
- 删除
- 设为常用或取消常用
- 快速添加 Tag
- 快速移除 Tag

可选增强能力：

1. 查看最近访问记录。
2. 复制网站标题和 URL。
3. 在新窗口或当前窗口中切换打开。

## 八、编辑网站弹窗

编辑网站弹窗仅 admin 可用，用于维护网站信息和 Tag 关系。

基础字段：

- 图标样式
- 图片地址
- 标题
- 描述
- 默认 URL
- LAN URL
- 打开方式
- 更多选项
- 保存
- 删除

Tag 字段：

- 已绑定 Tag
- 选择 Tag
- 新建 Tag
- 移除 Tag

业务规则：

1. 一个网站可以绑定多个 Tag。
2. 网站允许没有 Tag，但会进入 `未分类`。
3. 输入不存在的 Tag 时，admin 可创建新 Tag。
4. 删除网站时需要二次确认。
5. 游客不能打开编辑弹窗。

## 九、Tag 管理

Tag 管理仅 admin 可用。

管理能力：

1. 新建 Tag。
2. 修改 Tag 名称。
3. 修改 Tag 图标。
4. 修改 Tag 颜色。
5. 调整 Tag 排序。
6. 设置默认 Tag。
7. 禁用或启用 Tag。
8. 删除 Tag。
9. 查看关联网站数量。

删除规则：

1. 删除 Tag 不删除网站。
2. 删除后，网站仅移除该 Tag 关系。
3. 如果网站没有剩余 Tag，则进入 `未分类`。
4. 删除 Tag 需要二次确认。

## 十、常用与最近访问

### 10.1 常用网站

1. admin 可将网站设为常用。
2. admin 可取消常用。
3. 常用网站对游客可见。
4. `常用` 可作为系统内置视图展示。

### 10.2 最近访问

1. 最近访问存储在浏览器本地。
2. 游客模式和 admin 模式均不向服务端写入最近访问记录。
3. 最近访问不改变网站本身 Tag。
4. `最近访问` 可作为系统内置视图展示。
5. 清理浏览器本地数据后，最近访问记录可丢失。

## 十一、批量管理

批量管理仅 admin 可用。

批量能力：

1. 批量给网站添加 Tag。
2. 批量从网站移除 Tag。
3. 批量删除网站。
4. 调整网站展示顺序。

业务规则：

1. 批量添加 Tag 时，已存在的绑定关系不能重复创建。
2. 批量移除 Tag 时，未绑定该 Tag 的网站不应报错。
3. 批量删除网站需要二次确认。
4. 批量删除网站后，需要同步清理网站与 Tag、常用状态的关联数据。
5. 调整网站展示顺序后，首页和筛选结果需要按新的排序值展示。

## 十二、导入导出

导入导出仅 admin 可用。

导出能力：

1. 支持导出全部网站和 Tag 配置。
2. 支持选择指定网站导出。
3. 支持选择指定 Tag 导出。
4. 导出内容需要包含网站基础信息、Tag 信息、网站与 Tag 的绑定关系、常用状态和排序值。
5. 导出不包含 admin 密码和 Session 信息。

导入能力：

1. 支持导入网站和 Tag 配置。
2. 导入前需要校验数据格式。
3. 导入时需要处理 Tag 名称冲突。
4. 导入时需要处理网站 URL 或标题冲突。
5. 导入成功后需要保持网站与 Tag 绑定关系一致。
6. 导入失败时不能产生半成功数据。

## 十三、搜索与筛选

搜索范围：

- 标题
- 描述
- 默认 URL
- LAN URL
- Tag 名称

组合规则：

1. 搜索关键字与 Tag 筛选可同时生效。
2. 搜索结果只在当前 Tag 筛选范围内过滤。
3. 清空搜索后恢复当前 Tag 筛选结果。
4. 多 Tag 筛选始终使用“同时包含所有选中 Tag”的规则。
5. 游客和 admin 均可使用搜索与筛选。

## 十四、数据要求

基础数据要求：

1. 网站和 Tag 之间是多对多关系。
2. 同一个网站不能重复绑定同一个 Tag。
3. Tag 名称不能重复。
4. 默认 Tag 不存在或不可用时，需要回退到 `全部` 视图。
5. 常用状态属于网站全局配置，由 admin 管理。
6. 网站排序值由 admin 管理。
7. admin 密码需要加密存储，不能明文持久化。
8. 最近访问数据存储在浏览器本地，不进入服务端数据模型。

数据一致性要求：

1. 删除 Tag 时需要同步删除网站与 Tag 的关联关系。
2. 删除网站时需要同步删除网站与 Tag、常用状态相关的关联数据。
3. 写入 Tag 关系时需要具备幂等能力，重复提交不能产生重复绑定关系。
4. 保存网站信息失败时，不能产生半成功的 Tag 绑定数据。
5. 游客模式不能产生服务端数据写入。
6. 导入配置失败时，不能产生半成功数据。
7. 修改 admin 密码成功后，后续登录必须使用新密码。

## 十五、权限要求

系统权限按游客和 admin 两种访问方式划分。

| 操作 | 游客 | admin |
| --- | --- | --- |
| 查看网站 | 支持 | 支持 |
| 打开网站 | 支持 | 支持 |
| 复制链接 | 支持 | 支持 |
| 搜索网站 | 支持 | 支持 |
| 使用 Tag 筛选 | 支持 | 支持 |
| 查看常用网站 | 支持 | 支持 |
| 查看最近访问 | 支持 | 支持 |
| 登录 admin | 不支持 | 支持 |
| 退出 admin | 不支持 | 支持 |
| 修改 admin 密码 | 不支持 | 支持 |
| 新增网站 | 不支持 | 支持 |
| 编辑网站 | 不支持 | 支持 |
| 删除网站 | 不支持 | 支持 |
| 批量删除网站 | 不支持 | 支持 |
| 调整网站顺序 | 不支持 | 支持 |
| 上传图标 | 不支持 | 支持 |
| 新增 Tag | 不支持 | 支持 |
| 编辑 Tag | 不支持 | 支持 |
| 删除 Tag | 不支持 | 支持 |
| 批量添加 Tag | 不支持 | 支持 |
| 批量移除 Tag | 不支持 | 支持 |
| 设置默认 Tag | 不支持 | 支持 |
| 设置常用网站 | 不支持 | 支持 |
| 导出配置 | 不支持 | 支持 |
| 导入配置 | 不支持 | 支持 |

游客模式下，不展示编辑、删除、保存、上传、Tag 管理等管理入口。

所有管理类接口必须校验有效 admin Session，不能只依赖前端隐藏入口。

## 十六、非功能需求

1. 页面加载性能不能因 Tag 数量增加明显下降。
2. Tag 筛选响应应即时完成。
3. admin 执行编辑、删除、保存操作时需要有明确反馈。
4. 删除 Tag 和删除网站需要二次确认。
5. 网站、Tag、常用、最近访问数据需要保持一致。
6. 移动端或小屏幕下，Tag 区域需要可滚动或折叠。
7. 游客模式不能暴露管理接口入口。
8. 游客模式不能产生服务端写入。
9. 日志信息统一使用英文。
10. 初始 admin 密码只能在未配置密码时生成并打印。
11. admin Session 需要设置有效期。
12. 导入、批量删除等高风险操作需要保证事务一致性。

## 十七、验收标准

1. 首页默认进入默认 Tag 视图。
2. 网站可以绑定多个 Tag。
3. 多 Tag 筛选结果必须同时包含所有选中 Tag。
4. Tag 支持颜色、图标、排序。
5. 游客可以查看、搜索、筛选、打开网站。
6. 游客不能新增、编辑、删除网站或 Tag。
7. 游客界面不展示管理入口。
8. admin 可以管理网站、Tag、默认 Tag 和常用网站。
9. 网站卡片打开、编辑、删除、图标配置功能在 admin 模式下正常。
10. 删除 Tag 不会删除网站。
11. 无 Tag 网站可以在 `未分类` 中找到。
12. 默认 Tag 不存在或不可用时，系统能回退到 `全部` 视图。
13. 搜索关键字与 Tag 筛选可以同时生效。
14. 首次启动且未配置 admin 密码时，系统能生成随机密码并打印英文日志。
15. admin 可以修改密码，修改后新密码生效。
16. 管理接口必须基于有效 Session 才能调用。
17. admin 可以导出全部配置或指定网站、指定 Tag 的配置。
18. admin 可以导入配置，导入失败时不能产生半成功数据。
19. 最近访问记录存储在浏览器本地，不写入服务端。
20. admin 可以批量添加 Tag、批量移除 Tag、批量删除网站。
21. admin 可以调整网站展示顺序。

## 十八、待进一步确认事项

1. 导入导出文件格式使用 JSON、YAML，还是同时支持两者。
2. admin Session 有效期设置为多久。
3. 是否需要提供 Tag 合并能力。
4. 导入时遇到网站冲突时，采用跳过、覆盖，还是生成副本。
5. 随机初始密码长度和复杂度规则。
