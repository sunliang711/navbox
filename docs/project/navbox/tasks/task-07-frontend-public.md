# 任务 07：前端首页与游客体验

## 目标

实现游客可用的首页，包括 Tag 筛选、搜索、网站卡片和最近访问本地记录。

## 技术方案

1. React + TypeScript 实现 SPA。
2. 使用 API 获取网站和 Tag。
3. Tag 多选筛选由后端查询支持，前端维护选中状态。
4. 最近访问存储在浏览器 `localStorage`。
5. 卡片菜单游客模式只展示打开、复制链接等只读操作。

## 验收标准

1. 首页默认进入默认 Tag 视图。
2. 游客可以搜索网站。
3. 游客可以多选 Tag 筛选。
4. 游客可以打开默认 URL 和 LAN URL。
5. 最近访问保存在本地，不调用服务端写接口。
6. 游客界面不展示管理入口。

## 实现步骤

1. 创建 API Client。
2. 实现首页布局。
3. 实现 Tag 导航区。
4. 实现搜索。
5. 实现网站卡片。
6. 实现最近访问 localStorage。
7. 实现游客卡片菜单。

## 风险

1. 卡片视觉需要兼容不同图标和颜色。
2. 移动端 Tag 区域需要可滚动或折叠。
