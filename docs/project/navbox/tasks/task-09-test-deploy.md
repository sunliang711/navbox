# 任务 09：测试、验收与部署文档

## 目标

完成后端测试、前端构建验证、Docker 部署验证和使用说明。

## 技术方案

1. 后端使用 Go 单元测试和接口测试。
2. 前端执行 TypeScript 检查和构建。
3. Docker Compose 验证完整启动。
4. 编写 README 和部署说明。

## 验收标准

1. `go test ./...` 通过。
2. 前端构建通过。
3. Docker Compose 启动成功。
4. 首次启动能打印初始 admin 密码。
5. 浏览器能访问首页。
6. admin 登录后可完成核心管理操作。

## 实现步骤

1. 编写 Service 层单元测试。
2. 编写 Handler 层接口测试。
3. 执行前端构建检查。
4. 执行 Docker Compose 冒烟测试。
5. 更新 README。
6. 更新部署文档。

## 前置依赖

任务 06、任务 08。

## 风险

1. Docker 环境需要可访问镜像源。
2. 集成测试依赖 PostgreSQL，需要明确测试环境。
